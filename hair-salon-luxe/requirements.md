# 要件定義書 — hair salon LUXE

## 1. サイト概要

| 項目 | 内容 |
|---|---|
| サイト名 | hair salon LUXE |
| 種別 | 架空の美容室 ランディングページ（1ページ完結型） |
| 目的 | サロンの世界観・強みを伝え、予約・問い合わせへ誘導する |
| デザインコンセプト | ベージュ × ブラウンを基調とした、落ち着いた上品な雰囲気 |

---

## 2. ターゲット

- **性別**: 女性
- **年齢**: 20代〜40代
- **経営形態**: 個人経営サロン
- **想定ニーズ**: プライベート空間・品質重視・利便性を求めるユーザー

---

## 3. ページ構成

1ページ構成のランディングページ。各セクションはナビゲーションからスムーズスクロールで遷移する。

| # | セクション名 | ID |
|---|---|---|
| 1 | ヘッダー | — |
| 2 | ヒーロー | — |
| 3 | コンセプト | `#concept` |
| 4 | メニュー・料金表 | `#menu` |
| 5 | スタイリスト紹介 | `#stylist` |
| 6 | ご予約・お問い合わせ | `#contact` |
| 7 | フッター | — |

---

## 4. 各セクションの内容

### 4-1. ヘッダー
- サロン名ロゴ（"hair salon LUXE"）を左上に表示
- ナビゲーションメニュー（コンセプト・メニュー・スタイリスト・ご予約）を右側に表示
- ページ上部に固定表示（`position: fixed`）
- スクロール時に影を付与してヘッダーを強調
- モバイル時はハンバーガーボタンに切り替え、タップでオーバーレイメニューを開閉

### 4-2. ヒーロー
- キャッチコピー「あなただけの、上質な時間を」
- サロン名の大型タイポグラフィ表示
- サブテキスト「完全個室で、心ゆくまでくつろげるプライベートサロン」
- ご予約へ誘導するCTAボタン（`#contact` へスクロール）
- 背景はCSSグラデーションのみ（外部画像不使用）

### 4-3. コンセプト（3つの強み）
| 強み | 内容 |
|---|---|
| 完全個室のプライベート空間 | 他のお客様を気にせず過ごせる個室 |
| オーガニック製品を使用 | オーガニック認証製品のみ使用 |
| 駅徒歩3分の好アクセス | ○○駅から徒歩3分 |

- SVGアイコン + 見出し + 説明文のカード形式（3カラム）
- ホバー時に浮き上がるアニメーション

### 4-4. メニュー・料金表
カード形式（2カラム × 2行）で4メニューを掲載。

| メニュー | 掲載内容（例） |
|---|---|
| カット | カット / カット＋シャンプー / カット＋ブロー / 前髪カット |
| カラー | リタッチカラー / 全体カラー / ハイライト / グラデーション |
| パーマ | ショート / ミディアム / ロング / デジタルパーマ |
| トリートメント | システムTR / オーガニックTR / ヘッドスパ30分・60分 |

- 料金はすべて税込み表示
- 髪の長さ・状態により異なる旨の注意書きを記載

### 4-5. スタイリスト紹介
- **オーナースタイリスト**: 田中 美咲（Misaki Tanaka）
- プロフィール写真エリア（プレースホルダーとしてイニシャルアバターを使用）
- キャリア紹介テキスト
- 得意なスタイルタグ一覧（ナチュラルカラー・外国人風カラー・柔らかウェーブ・大人ショート・艶感スタイル・ヘッドスパ）
- 保有資格一覧

### 4-6. ご予約・お問い合わせフォーム
以下の入力項目を設ける。

| 項目 | 種別 | 必須 |
|---|---|---|
| お名前 | テキスト | ○ |
| メールアドレス | メール | ○ |
| 電話番号 | 電話番号 | ○ |
| ご希望メニュー | セレクトボックス | ○ |
| ご希望日時（第1希望） | 日時 | ○ |
| ご希望日時（第2希望） | 日時 | — |
| ご要望・ご質問 | テキストエリア | — |

- 必須項目に「必須」バッジを表示
- バリデーション: 未入力・メール形式不正を検知しエラー表示
- 入力修正時にリアルタイムでエラーを解除
- 送信後、フォームを非表示にして完了メッセージを表示

### 4-7. フッター
- サロン名ロゴ
- 住所（〒150-0001 東京都渋谷区神宮前1-1-1 LUXEビル2F）
- 電話番号（03-0000-0000）
- 営業時間（平日 10:00〜20:00 / 土日祝 9:00〜19:00）
- 定休日（毎週火曜日）
- ナビゲーションリンク
- コピーライト表記

## 5. 技術要件

### 使用技術
| 種別 | 技術 |
|---|---|
| マークアップ | HTML5 |
| スタイリング | CSS3（CSSカスタムプロパティ使用） |
| スクリプト | Vanilla JavaScript（ES6+） |
| 外部ライブラリ | 不使用 |
| 外部画像 | 不使用（すべてCSS・SVGで代替） |

### コーディング規約
- コメントは日本語で記述する
- ファイル名はケバブケースを使用する（例: `style.css`、`script.js`）

### レスポンシブ対応
| ブレークポイント | 対象 | 主な変更内容 |
|---|---|---|
| 〜 1024px | タブレット | スタイリストセクションのレイアウト調整 |
| 〜 768px | タブレット〜モバイル | ナビ→ハンバーガー切替、各カードを縦並び、フォームを1カラム |
| 〜 480px | スマートフォン | 余白・フォントサイズの最終調整 |

### JavaScript 機能
| 機能 | 実装方法 |
|---|---|
| スムーズスクロール | CSS `scroll-behavior: smooth` |
| ハンバーガーメニュー | クラスの付け外しでオーバーレイを表示 |
| スクロールアニメーション | `IntersectionObserver` でフェードイン |
| フォームバリデーション | `submit` イベントで入力値を検証 |
| リアルタイムエラー解除 | `input` イベントでエラー状態を解除 |

## 6. 対応範囲と対応範囲外

### 対応範囲（実装済み）
- 全セクションのHTML構造・デザイン
- ベージュ × ブラウンを基調としたビジュアルデザイン
- PCおよびスマートフォン・タブレットへのレスポンシブ対応
- ハンバーガーメニューによるモバイルナビゲーション
- スクロール連動のフェードインアニメーション
- 予約フォームの見た目・バリデーション・送信完了演出

### 対応範囲外（未実装）
- フォームのメール送信機能（バックエンド処理）
- CMS・管理画面
- 実際の店舗写真・スタイリスト写真の掲載
- 多言語対応
- SNS連携（Instagram埋め込み等）
- オンライン予約システムとの連携
- アクセス解析（Google Analytics等）
- SEO対策（meta description・OGP等の詳細設定）
